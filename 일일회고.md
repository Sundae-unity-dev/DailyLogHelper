# 일일회고록 정리 도우미

학생이 그날 작업한 메모를 받아 Unity 3기 과제 양식 PDF로 변환한다.

입력: $ARGUMENTS

## 0단계 — 경로 확인 및 규칙 읽기

아래 명령으로 스킬 경로를 확인한다:
```
echo "$HOME/.claude/DailyLogHelper"
```

`$HOME/.claude/DailyLogHelper/rules.md` 를 읽어 정리 규칙을 숙지한다:
```
cat "$HOME/.claude/DailyLogHelper/rules.md"
```

## 1단계 — 모드 분기

`$ARGUMENTS` 값에 따라 모드를 결정한다.

- **`paste` 로 시작하면 (예: `/일일회고 paste`)** → **일괄 모드**
  - "오늘 작업한 내용을 자유롭게 한 번에 붙여넣어 주세요. 두서없어도 괜찮습니다. 이름·날짜·몇차 프로젝트인지도 같이 적어주시면 좋아요." 라고 안내.
  - 학생이 내용을 던지면 rules.md 규칙대로 분류하여 JSON 생성.
  - 빈 섹션이 있으면 그대로 비워두고 PDF 생성. 강제로 되묻지 않음.
  - `project_order`가 메모에 없으면 한 번만 짧게 되묻는다.

- **빈 인자거나 다른 텍스트** → **대화 모드**
  - 다음 순서로 한 번에 한 가지씩만 묻는다. 한 번에 여러 질문 금지.
  - 학생이 "없음" / "안 함" / 엔터만 치면 그 섹션은 빈 문자열로 둔다.
  - 학생이 메모를 던지면 rules.md 규칙대로 **한 단락(2~4줄)으로 살을 붙여** 다음 질문으로 넘어간다 (사실 범위 안에서만).
  - 질문 순서:
    1. 📛 이름과 날짜 (예: "홍길동 / 2025-11-13"). 날짜 생략 시 오늘.
    2. 🔢 **몇 차 프로젝트인가요?** (예: `2차`, `최종`, `미니`). 답한 값을 `project_order`로 저장.
    3. 📝 **Scrum** — 오늘 한 일 / 진행 중 / 내일 계획. (없으면 "스킵")
    4. 🛠 **Project** — 오늘 본인이 기여한 부분. 스크린샷 경로 있으면 같이.
    5. 🧯 **Trouble Shooting** — 막힌 문제와 해결 과정. (없으면 "스킵")
    6. 🔗 **링크** — 깃헙 / 블로그 / 학습 링크. (없으면 "스킵")

- `$ARGUMENTS` 가 비어 있지 않고 `paste`도 아니면 그 텍스트를 **이미 받은 메모로 간주**하고
  일괄 모드로 동작. 부족한 정보(특히 `project_order`)는 한두 가지만 짧게 되묻는다.

## 2단계 — JSON 작성

수집한 내용을 rules.md 의 JSON 스키마에 맞게 정리한다.
- 순수 JSON 객체만 (코드블록 래퍼 없이).
- 한글 그대로 (이스케이프 금지).
- 모든 본문 필드는 **한 단락 이상 살을 붙여 작성**. 단, 사실 범위를 벗어나지 말 것.
- 이미지 경로가 섞여 있으면 rules.md 의 "이미지 첨부 처리" 표에 따라 해당 섹션 본문 끝에 `![설명](경로)` 로 삽입.

저장 경로:
```
echo "$HOME/.claude/DailyLogHelper/_temp_log.json"
```
Write 도구로 위 경로에 JSON 을 UTF-8 로 저장한다.

## 3단계 — PDF 렌더링

아래 명령을 실행한다. (출력 디렉터리는 자동으로 `~/Desktop/일일회고/` 생성)

**Windows / Git Bash:**
```
node "$HOME/.claude/DailyLogHelper/render.js" --json "$HOME/.claude/DailyLogHelper/_temp_log.json" --template "$HOME/.claude/DailyLogHelper/template.html"
```

성공 시 stdout 에 PDF 절대 경로와 HTML(검토용) 경로가 출력된다.

## 4단계 — 결과 보고

학생에게 친절하게 한 단락으로 보고한다:
- 생성된 PDF 절대 경로
- 파일명 (`yymmdd_{project_order} 프로젝트_Unity 3기_OOO.pdf` 패턴 확인)
- 검토용 HTML 경로 (선택)
- 다음 행동 안내: "이 PDF 를 그대로 과제 제출란에 업로드하시면 됩니다."

## 오류 처리

- `puppeteer` 미설치 → 학생에게:
  ```
  cd ~/.claude/DailyLogHelper
  npm install
  ```
  실행하라고 안내. 그 후 다시 `/일일회고` 실행 권유.
- Node.js 미설치 → https://nodejs.org 에서 LTS 설치 후 재시도 안내.
- 이미지 경로가 깨졌거나 (예: 한글 경로 + 특수문자) 렌더 실패 → 경로를 따옴표로 감싸 다시 확인 요청.
- 그 외 오류는 stderr 메시지를 학생에게 그대로 보여주고 원인 추정.
